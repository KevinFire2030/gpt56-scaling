# 자체 검증 결과

검증 대상: `runs/test1a/luna_l/receipts.json`
입력 폴더: `입력데이터/영수증/`

## 검증 항목

- 입력 이미지: R01.png~R20.png 총 20개 확인
- JSON 최상위 구조: 배열 확인
- 영수증 객체 수: 20개
- `receipt_id`: R001~R020 순서 및 중복 없음
- 각 영수증 필수 필드: `receipt_id`, `store_name`, `date`, `items`, `subtotal`, `tax`, `total`, `flags` 모두 존재
- 각 `items` 원소 필수 필드: `name`, `qty`, `unit_price`, `amount` 모두 존재
- 금액 필드: KRW 정수 또는 `null`만 사용
- `flags`: 배열이며 문제 설명은 한국어로 기록
- JSON 문법: Python `json.loads`로 정상 파싱

## 산술 검증 결과

- R001~R003, R005~R010, R012~R018, R020: 품목 합계/소계 및 소계+세금/합계 관계 확인
- R004: 영수증에 표시된 소계 25,000원, 부가세 2,500원, 합계 28,500원을 그대로 기록했으며, 합산 결과가 일치하지 않아 `flags`에 기록
- R011: 영수증에 표시된 품목 금액 합계 29,000원, 소계 27,000원, 부가세 2,900원, 합계 29,900원을 그대로 기록했으며, 품목 합계와 소계 불일치를 `flags`에 기록
- R007: 영수증 상단의 `금액단위: 천원`을 적용하여 45.0→45,000원, 90.0→90,000원, 98.0→98,000원 등으로 환산하고 환산 사실을 `flags`에 기록
- R019: 얼룩으로 가려진 감자튀김 수량은 `null`로 기록하고, 판독 가능한 단가/금액은 기록

모든 구조·개수·식별자·타입·파싱 검증 통과.
