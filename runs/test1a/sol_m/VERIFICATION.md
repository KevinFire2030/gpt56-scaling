# 영수증 JSON 자체 검증 결과

## 대상

- 입력 이미지: `입력데이터/영수증/R01.png` ~ `R20.png` (20장)
- 결과 파일: `runs/test1a/sol_m/receipts.json`
- 검증 시점: 결과 파일 생성 직후

## 판독 및 검증 방법

1. 20개 입력 이미지의 존재와 이미지 크기(각 1024×1536)를 확인했다.
2. 전체 연락처 시트와 개별 확대 이미지를 이용해 상호명, 날짜, 품목, 수량, 단가, 금액, 공급가액, 부가세, 합계를 시각적으로 교차 판독했다.
3. JSON 파서로 결과 파일을 다시 읽어 문법 유효성을 확인했다.
4. 객체 수, `receipt_id` 순서/중복, 필수 키, `items`의 필수 키, 금액 자료형을 프로그램으로 검사했다.
5. 각 품목의 `qty × unit_price = amount`, 품목 금액 합계와 `subtotal`, `subtotal + tax = total`을 비교했다.

## 구조 검증 결과

- JSON 문법: 통과
- 최상위 형식: 배열, 통과
- 영수증 객체 수: 20개, 통과
- `receipt_id`: `R001`부터 `R020`까지 순서대로 정확히 존재, 통과
- 중복 ID: 없음
- 필수 최상위 키: 모든 객체 통과
  - `receipt_id`, `store_name`, `date`, `items`, `subtotal`, `tax`, `total`, `flags`
- 품목 필수 키: 모든 품목 통과
  - `name`, `qty`, `unit_price`, `amount`
- 금액 자료형: 모든 판독 금액이 KRW 정수, 통과
- `null` 값: R019 감자튀김의 `qty` 1건
  - 이미지의 원형 얼룩으로 수량이 가려져 있어 추정하지 않고 `null`로 기록했다.

## 산술 검증 결과

- 일반 품목의 `qty × unit_price = amount`: 통과
- 자동 검사에서 확인된 원본 영수증 표기 불일치:
  - R004: `subtotal + tax = 27,500원`이나 영수증의 `total`은 `28,500원`으로 1,000원 차이.
  - R011: 품목 금액 합계는 29,000원이나 영수증의 `subtotal`은 27,000원으로 2,000원 차이.
- 추가 원본 표기 이상:
  - R011: 부가세 2,900원이 공급가액 27,000원의 10%와 일치하지 않음.
  - R017: 부가세 1,250원이 공급가액 22,500원의 10%인 2,250원과 일치하지 않음.
- 위 값들은 임의 보정하지 않고 이미지에 인쇄된 값을 그대로 기록했으며 각 객체의 `flags`에 한국어 설명을 남겼다.

## 특이사항

- R007은 영수증에 `금액단위: 천원`이라고 표시되어 있어 `45.0`을 45,000원처럼 모든 금액을 KRW 정수로 환산했다.
- 문제 또는 변환 설명이 있는 ID: R004, R007, R011, R017, R019.
- 그 외 영수증은 판독된 품목 금액 합계, 공급가액, 부가세, 합계 간 산술 관계가 일치했다.

## 최종 판정

요청된 스키마와 ID 범위를 만족하며 JSON 파싱 및 구조 검증을 통과했다. 원본 이미지 자체의 산술 불일치와 판독 불가 값은 보정·추정하지 않고 `flags`와 `null`로 명시했다.
