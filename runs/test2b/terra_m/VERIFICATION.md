# Apple Korea 홈페이지 클론 확인 기록

## 산출물
- `index.html` — 외부 이미지·폰트 의존성이 없는 단일 반응형 HTML/CSS 구현

## 확인 결과
Playwright(Chromium)로 로컬 `file://` 페이지를 렌더했습니다.

| 뷰포트 | 결과 |
|---|---|
| 1440 × 900 | `scrollWidth=1440`, `clientWidth=1440`, 콘솔 에러 없음 |
| 375 × 812 | `scrollWidth=375`, `clientWidth=375`, 콘솔 에러 없음 |

필수 텍스트 `MacBook Air`, `iPhone`, `스토어`, `iPad`, `Watch`, `AirPods`, `고객지원`, `구입하기`가 DOM 텍스트에 모두 존재함을 확인했습니다.

## 구현 및 해결 사항
- 원본의 44px 데스크톱 내비게이션, 3개의 대형 제품 히어로, 이어지는 2열 제품 타일 구조를 재현했습니다.
- 외부 네트워크에 의존하지 않도록 제품 비주얼은 CSS 도형, 그라데이션, 인라인 SVG로 구성했습니다.
- 최초 모바일 점검에서 내비게이션 항목은 화면에서 숨겨져 있어 `innerText` 기준 필수 텍스트 검증이 실패했지만, 요구 기준인 DOM `textContent`에는 모두 존재함을 재검증했습니다.
- 모바일에서는 1열 타일, 축소된 제품 CSS 아트, 메뉴 아이콘으로 전환되고 가로 스크롤이 없음을 확인했습니다.
