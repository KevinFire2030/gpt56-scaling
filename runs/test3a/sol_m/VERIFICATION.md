# DANTE LABS — 구현 및 검증 기록

## 실행 방법

이 폴더에서 아래 명령을 실행한 뒤 브라우저로 접속합니다.

```bash
python -m http.server 8765 --bind 127.0.0.1
```

- URL: `http://127.0.0.1:8765/index.html`
- 조작: `RUN EXPERIMENT` 버튼 또는 `Space`
- 초기화: `RESET` 버튼 또는 `R`
- 시점 전환: `VIEW +`
- 수평 시점 조정: 3D 화면을 좌우로 드래그

> CDN과 Astronaut GLB를 사용하므로 최초 실행 시 인터넷 연결이 필요합니다. GLB 로딩 실패 시 페이지에 내장된 절차형 우주비행사 모델로 자동 대체됩니다.

## 물리 모델과 기대 측정값

양쪽 모두 같은 초기 수직 속도 `v₀ = 4.5 m/s`를 사용하며 공기저항은 제외합니다.

- 높이: `h(t) = max(0, v₀t - ½gt²)`
- 최고점: `h_max = v₀² / (2g)`
- 총 체공시간(이륙부터 같은 높이에 착지): `T = 2v₀ / g`

| 환경 | 중력가속도 | 이론 최고점 | 이론 체공시간 | UI 반올림 표시 |
|---|---:|---:|---:|---|
| EARTH | 9.81 m/s² | 1.032110 m | 0.917431 s | `1.03 m`, `0.92 s` |
| MOON | 1.62 m/s² | 6.250000 m | 5.555556 s | `6.25 m`, `5.56 s` |

`FALL_TIME_S`라는 UI 필드명은 요청을 그대로 따르되, 값은 비교가 직관적인 **전체 점프 체공시간**(launch → landed)을 뜻합니다. 순수 최고점→착지 시간은 위 값의 절반입니다.

## 실제 검증 결과

로컬 HTTP 서버와 브라우저에서 직접 실행했습니다.

### 로딩/정적 검증

- `index.html` UTF-8 로딩: PASS
- inline JavaScript `node --check`: PASS
- Three.js r128 CDN HEAD: HTTP 200
- GLTFLoader r128 CDN HEAD: HTTP 200
- Three.js WebGL 장면 2개 렌더링: PASS
- Astronaut.glb 양쪽 렌더링: PASS
- 브라우저 JavaScript 오류: **0개**
- 브라우저 콘솔 오류: **0개**

### 상호작용 실측

브라우저에서 `RUN EXPERIMENT`를 클릭하고 양쪽 착지 완료 후 DOM과 시뮬레이션 상태를 읽었습니다.

```text
running: false
elapsed: 5.5895 s (렌더 프레임 기반 완료 감지 시각)

[EARTH] jump_apex_m = 1.03 m
[EARTH] fall_time_s = 0.92 s
[EARTH] status      = LANDED

[MOON]  jump_apex_m = 6.25 m
[MOON]  fall_time_s = 5.56 s
[MOON]  status      = LANDED
```

최종 텔레메트리는 이론값과 일치했습니다. `elapsed`는 전체 실험 루프가 달 착지를 감지한 실제 렌더 프레임 시각이므로 이론값 5.555556초보다 한 프레임 단위로 클 수 있지만, 각 환경의 `fall_time_s`는 정확한 물리 이벤트 시각으로 고정됩니다.

### 시각 검수

- 초기 상태와 실험 완료 상태를 각각 브라우저 스크린샷으로 확인했습니다.
- 두 우주비행사, 지면, 높이 눈금, 중앙 구분선, VS 배지, 상태 배너가 정상 표시됐습니다.
- 하단 텔레메트리의 겹침/잘림 없이 최종값이 정렬됐습니다.
- 로딩 오버레이는 모델 준비 후 사라졌고 빈 캔버스가 남지 않았습니다.

## 60fps 최적화 사항

- 하나의 `requestAnimationFrame` 루프와 하나의 `WebGLRenderer`만 사용합니다.
- 좌/우 장면은 `setViewport` + `setScissor`로 같은 프레임에 렌더링합니다.
- 기기 픽셀 비율을 최대 `1.75`로 제한해 고해상도 화면의 과도한 GPU 부하를 줄였습니다.
- 그림자 맵은 장면별 1024², PCF Soft Shadow로 제한했습니다.
- 정적 지형/별/크레이터는 생성 후 재사용하며 프레임마다 지오메트리를 만들지 않습니다.
- 프레임 지연 시 물리 적분 입력을 최대 35ms로 제한합니다.

실제 fps는 GPU, 브라우저, 화면 해상도에 따라 달라지지만, 코드 경로는 일반 데스크톱에서 60fps를 목표로 구성했습니다.
