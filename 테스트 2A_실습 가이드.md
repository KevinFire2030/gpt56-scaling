# 테스트 2A 실습 가이드: Hermes Agent로 함정 매출 분석하기

이 문서는 **테스트 2A: 함정 매출 데이터 분석 보고서**를 Hermes Agent로 따라 하는 실습 가이드입니다.

목표는 `sales.csv`를 분석해서 데이터 품질 문제를 찾고, 경영 회의용 보고서를 만드는 것입니다.

중요: 실습은 Codex CLI에서 직접 하는 것이 아니라 **Hermes Agent 프로파일**로 실행합니다.

## 목표

입력:

```text
입력데이터/함정매출_sales.csv
```

출력:

```text
runs/test2a/terra_h/report.md
runs/test2a/terra_h/analysis.py
runs/test2a/terra_m/report.md
runs/test2a/terra_m/analysis.py
...
```

채점기는 각 실행 폴더에서 아래 파일을 찾습니다.

```text
report.md
analysis.py
```

## 비교 조합

단테랩스 원문 기준 테스트 2A 비교 조합은 아래 6개입니다.

| 조합 | 프로파일 | model | reasoning |
| --- | --- | --- | --- |
| Terra High | `terra_h` | `openai/gpt-5.6-terra` | `high` |
| Terra Medium | `terra_m` | `openai/gpt-5.6-terra` | `medium` |
| Terra Ultra | `terra_u` | `openai/gpt-5.6-terra` | `ultra` |
| Luna Max | `luna_max` | `openai/gpt-5.6-luna` | `max` |
| Sol Medium | `sol_m` | `openai/gpt-5.6-sol` | `medium` |
| Terra Max | `terra_max` | `openai/gpt-5.6-terra` | `max` |

## 전체 계획

1. 입력 CSV가 있는지 확인합니다.
2. 6개 Hermes 프로파일을 준비합니다.
3. 각 프로파일이 기본 Codex 인증을 재사용하도록 설정합니다.
4. 조합별 실행 폴더를 만들고 `sales.csv`를 복사합니다.
5. 각 Hermes Agent로 `report.md`와 `analysis.py`를 생성합니다.
6. 채점기 `grade_test2a.py`를 실행합니다.
7. 사용 토큰, 실행시간, 추정 비용을 모아 최종 결과표를 작성합니다.
8. 테스트 중 나온 문제와 해결 방법을 기록합니다.

## 1. 테스트 2A가 하는 일 이해하기

이 테스트는 단순 합계가 아니라 **데이터 품질 점검 + 분석 보고서 작성**입니다.

에이전트가 해야 할 일:

1. `sales.csv`를 읽습니다.
2. 데이터 품질 문제를 찾습니다.
3. 문제를 보정하거나 주의하면서 인사이트를 냅니다.
4. 보고서 맨 앞에 요약 표를 넣습니다.
5. `report.md`와 `analysis.py`를 저장합니다.
6. `analysis.py`가 실행되는지 스스로 확인합니다.

주의: 데이터 안에는 여러 함정이 들어 있지만, 실습 프롬프트에는 함정 종류를 알려주지 않습니다. 모델이 스스로 찾아야 합니다.

## 6. 순차 실행하기

처음에는 병렬보다 순차 실행이 안전합니다.

```powershell
$profiles = "terra_h","terra_m","terra_u","luna_max","sol_m","terra_max"
foreach ($p in $profiles) {
  $start = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling\runs\test2a\$p"
  hermes -p $p --yolo -z $basePrompt --usage-file ".\usage.json"
  $end = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  [pscustomobject]@{
    profile=$p
    start=$start.ToString("o")
    end=$end.ToString("o")
    seconds=[math]::Round(($end-$start).TotalSeconds,1)
    report=(Test-Path ".\runs\test2a\$p\report.md")
    analysis=(Test-Path ".\runs\test2a\$p\analysis.py")
    usage=(Test-Path ".\runs\test2a\$p\usage.json")
  } | ConvertTo-Json -Compress | Tee-Object -FilePath ".\runs\test2a\run_times.jsonl" -Append
}
```

## 8. 채점기 실행

```powershell
$env:PYTHONUTF8 = "1"
$profiles = "terra_h","terra_m","terra_u","luna_max","sol_m","terra_max"
foreach ($p in $profiles) {
  Write-Host "==== $p ===="
  python .\reproduction-kit\reproduction-kit\graders\grade_test2a.py --run-dir ".\runs\test2a\$p"
}
```

채점 항목:

| 항목 | 뜻 |
| --- | --- |
| `summary_table` | 보고서 앞부분에 요약 표가 있는지 |
| `analysis_py_runs` | 분석 코드가 에러 없이 실행되는지 |
| `trap_recall` | 함정 발견 개수 |
| `traps_found` | 어떤 함정을 찾았는지 |
| `bait_T7` | 기온/날씨 미끼를 인과로 착각했는지 |
| `score_auto` | 자동 점수 |
| `evaluator_pass` | 통과 여부 |

중요: `grade_test2a.py`는 키워드 기반 보조 채점기입니다. 최종 판정은 `report.md`를 사람이 읽어 수동 확정할 수 있습니다.

## 9. 실패했을 때 고치는 방법

### report.md 없음

- Hermes가 보고서를 채팅으로만 답하고 파일로 저장하지 않은 경우입니다.
- 프롬프트 첫 줄에 “작업을 지금 실제로 수행하세요. 설명만 하지 마세요.”를 넣습니다.
- `.bat` alias 대신 `hermes -p <profile> --yolo -z ...`로 실행합니다.

### analysis.py 없음 또는 실행 실패

- 보고서만 만들고 분석 코드를 저장하지 않은 경우입니다.
- `analysis.py`를 반드시 저장하라고 다시 지시합니다.
- 실행 폴더 안에서 `python analysis.py`가 되는지 확인합니다.

### 채점기에서 recall이 낮음

- `report.md`에 데이터 품질 문제를 번호 목록으로 명확히 적었는지 확인합니다.
- 문제 위치, 기간, 수치를 함께 적어야 키워드 채점기가 잘 잡습니다.
- “기온 때문에 매출 증가”처럼 인과를 단정하면 오탐 처리될 수 있습니다.

### Windows 한글/UTF-8 오류

- 채점 전 아래를 설정합니다.

```powershell
$env:PYTHONUTF8 = "1"
```
