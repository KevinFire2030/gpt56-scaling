# 테스트 1A 실습 가이드: Hermes Agent 5개 프로파일로 동시에 실행하기

이 문서는 **테스트 1A: 영수증 이미지 구조화 추출**을 Hermes Agent로 따라 하는 실습 가이드입니다.

중요한 원칙은 두 가지입니다.

- Codex CLI가 아니라 **Hermes Agent**로 실행합니다.
- 모델+추론 조합마다 Hermes 프로파일을 따로 만들고, 5개 에이전트를 동시에 호출합니다.

## 목표

영수증 이미지 20장을 읽어 조합별로 `receipts.json`을 만듭니다.

입력:

```text
입력데이터/영수증/R01.png
...
입력데이터/영수증/R20.png
```

출력 예:

```text
runs/test1a/sol_m/receipts.json
runs/test1a/terra_m/receipts.json
runs/test1a/luna_m/receipts.json
runs/test1a/luna_h/receipts.json
runs/test1a/luna_l/receipts.json
```

## 전체 계획

1. Hermes가 설치되어 있는지 확인합니다.
2. 모델+추론 조합별 프로파일 5개를 만듭니다.
3. 각 프로파일에서 provider/model/추론 설정을 저장합니다.
4. 각 프로파일 alias를 만듭니다.
5. 결과 저장 폴더를 5개 만듭니다.
6. PowerShell Job으로 5개 에이전트를 동시에 실행합니다.
7. `receipts.json`이 생겼는지 확인합니다.
8. 정답 키를 준비합니다.
9. 채점기를 5개 폴더에 대해 실행합니다.
10. 최종 결과 표를 작성합니다.

## 3. 각 프로파일에 모델과 추론 설정 저장하기

각 프로파일을 선택한 뒤 `hermes model`을 실행해 해당 조합에 맞는 provider/model/추론 설정을 저장합니다.

```powershell
hermes profile use sol_m
hermes model

hermes profile use terra_m
hermes model

hermes profile use luna_m
hermes model

hermes profile use luna_h
hermes model

hermes profile use luna_l
hermes model
```

각 `hermes model` 화면에서 아래 의미에 맞게 선택합니다.

| 프로파일 | 선택할 조합 |
| --- | --- |
| `sol_m` | Sol Medium |
| `terra_m` | Terra Medium |
| `luna_m` | Luna Medium |
| `luna_h` | Luna High |
| `luna_l` | Luna Low |

인증 오류가 나면 OpenRouter API 키를 새로 넣는 대신, 기본 Codex 프로파일의 인증을 Hermes로 복사해 사용합니다.

이 실습에서는 각 Hermes 프로파일의 LLM 인증을 **기본 Codex 인증 재사용 방식**으로 맞춥니다.

```powershell

```

프로파일별 provider/model/reasoning 설정은 아래처럼 맞춥니다.

| 프로파일 | provider | model | reasoning |

| `luna_m` | `openai-codex` | `openai/gpt-5.6-luna` | `medium` |

## 4. alias 만들기

동시에 실행할 때는 `hermes profile use ...`로 기본 프로파일을 계속 바꾸면 안 됩니다. 병렬 실행 중 설정이 꼬일 수 있습니다.

대신 각 프로파일 alias를 만듭니다.

hermes profile alias luna_m

## 7. 5개 에이전트 동시에 실행하기

```powershell
$jobs = @()

$jobs += Start-Job -Name sol_m -ScriptBlock {
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  $runDir = "runs/test1a/sol_m"
  $prompt = $using:basePrompt.Replace('$runDir', $runDir)
  sol_m -z $prompt --usage-file ".\$runDir\usage.json"
}

$jobs += Start-Job -Name terra_m -ScriptBlock {
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  $runDir = "runs/test1a/terra_m"
  $prompt = $using:basePrompt.Replace('$runDir', $runDir)
  terra_m -z $prompt --usage-file ".\$runDir\usage.json"
}

$jobs += Start-Job -Name luna_m -ScriptBlock {
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  $runDir = "runs/test1a/luna_m"
  $prompt = $using:basePrompt.Replace('$runDir', $runDir)
  luna_m -z $prompt --usage-file ".\$runDir\usage.json"
}

$jobs += Start-Job -Name luna_h -ScriptBlock {
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  $runDir = "runs/test1a/luna_h"
  $prompt = $using:basePrompt.Replace('$runDir', $runDir)
  luna_h -z $prompt --usage-file ".\$runDir\usage.json"
}

$jobs += Start-Job -Name luna_l -ScriptBlock {
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  $runDir = "runs/test1a/luna_l"
  $prompt = $using:basePrompt.Replace('$runDir', $runDir)
  luna_l -z $prompt --usage-file ".\$runDir\usage.json"
}

$jobs | Wait-Job
$jobs | Receive-Job
```

각 에이전트는 자기 프로파일 설정으로 실행되고, 자기 폴더에 결과를 저장합니다.

## 11. 실패했을 때 고치는 방법

실패해도 괜찮습니다. 아래 순서로 고치면 됩니다.

### JSON 파싱 실패

- 파일 맨 앞은 `[`로 시작해야 합니다.
- 파일 맨 뒤는 `]`로 끝나야 합니다.
- ```json 같은 코드블록 표시는 제거합니다.

### 배열 20개가 아님

- `receipt_id`가 `R001`부터 `R020`까지 있는지 확인합니다.
- 빠진 번호를 찾아 해당 조합의 `receipts.json`만 고칩니다.

### 정확도가 낮음

- 틀린 영수증 번호를 확인합니다.
- 해당 PNG를 다시 보면서 값을 고칩니다.
- 금액은 쉼표 없이 정수로 저장합니다.

### flags가 이상함

- 합계, 공급가액, 부가세를 다시 계산합니다.
- 읽을 수 없는 값은 `null`로 두고 이유를 flags에 씁니다.

수정한 뒤 해당 조합만 다시 채점합니다.

```powershell
python .\reproduction-kit\reproduction-kit\graders\grade_test1a.py --run-dir .\runs\test1a\sol_m
```

## 13. 실제 테스트 과정 기록

이번 로컬 실습에서는 아래 순서로 진행했습니다.

1. `sol_m`, `terra_m`, `luna_m`, `luna_h`, `luna_l` Hermes 프로파일을 만들었습니다.
2. 각 프로파일의 provider를 `openai-codex`로 맞추고, 기본 Codex 인증을 Hermes 인증 저장소로 복사했습니다.
3. 각 프로파일에 Sol/Terra/Luna 모델과 Medium/High/Low 추론 설정을 저장했습니다.
4. 각 프로파일 alias를 만들어 `sol_m`, `terra_m`, `luna_m`, `luna_h`, `luna_l` 명령처럼 호출할 수 있게 했습니다.
5. 각 프로파일로 `receipts.json`을 생성했습니다.
6. 정답 키를 생성한 뒤 `grade_test1a.py`로 5개 결과를 채점했습니다.
7. Hermes 세션 로그에서 토큰 사용량과 실행 시간을 뽑아 최종 표에 반영했습니다.

## 14. 나왔던 문제와 해결 방법

| 문제 | 원인 | 해결 방법 |
| --- | --- | --- |
| OpenRouter provider 인증 오류 | `OPENROUTER_API_KEY`가 설정되어 있지 않았습니다. | OpenRouter 대신 기본 Codex 인증을 Hermes로 복사하고 provider를 `openai-codex`로 바꿨습니다. |
| 5개 병렬 실행 결과 추적이 불안정함 | PowerShell `Start-Job`으로 실행한 Hermes 작업의 세션/출력 추적이 끊기는 경우가 있었습니다. | 실제 실측은 프로파일별로 순차 실행해 안정적으로 `receipts.json`과 세션 로그를 확보했습니다. |
| Hermes 프로세스가 파일 생성 후 종료되지 않음 | 결과 파일은 만들어졌지만 프로세스가 계속 대기하는 경우가 있었습니다. | `receipts.json` 생성과 세션 저장을 확인한 뒤 남은 Hermes 프로세스를 종료했습니다. |
| 정답 키 생성 시 한글 경로 인코딩 오류 | Windows PowerShell 기본 인코딩에서 `UnicodeEncodeError`가 났습니다. | `$env:PYTHONUTF8='1'`을 설정한 뒤 정답 키 생성 명령을 다시 실행했습니다. |
| `usage.json`이 비어 있거나 생성되지 않음 | Hermes가 정상 종료되지 않으면 `--usage-file` 기록이 남지 않을 수 있습니다. | Hermes 세션 export JSONL에서 토큰 필드를 읽어 총 토큰을 계산했습니다. |
| Codex 인증 사용 시 실제 과금 비용이 0으로 기록됨 | Codex 구독 인증 경로에서는 실행별 달러 비용이 직접 과금값으로 기록되지 않았습니다. | 비교용으로 OpenRouter 공개 단가 기준의 추정 비용을 별도 계산했습니다. |
