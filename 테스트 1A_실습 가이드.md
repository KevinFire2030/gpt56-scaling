# 테스트 1A 실습 가이드: Hermes Agent로 영수증 20장 JSON 만들기

이 문서는 **테스트 1A**를 처음 해 보는 사람도 따라 할 수 있게 만든 실습 가이드입니다.

중요한 점은 하나입니다.

**이 실습은 Codex CLI에서 하지 않고, Hermes Agent로 진행합니다.**

목표는 Hermes Agent에게 영수증 이미지 20장을 읽게 하고, `receipts.json` 파일을 만들게 한 뒤, 제공된 채점기로 결과를 확인하는 것입니다.

## 테스트 1A가 무엇인가요?

테스트 1A는 영수증 이미지를 읽고 필요한 정보를 JSON으로 정리하는 과제입니다.

입력:

```text
입력데이터/영수증/R01.png
입력데이터/영수증/R02.png
...
입력데이터/영수증/R20.png
```

출력:

```text
runs/test1a/hermes-test/receipts.json
```

`receipts.json` 안에는 영수증 20장에 대한 정보가 JSON 배열로 들어가야 합니다.

## 전체 계획

아래 순서대로 진행합니다.

1. Hermes 실습용 프로파일을 만든다.
2. 그 프로파일에서 사용할 provider와 model을 설정한다.
3. 테스트 1A 작업 폴더를 만든다.
4. Hermes Agent에게 영수증 20장을 분석하라고 시킨다.
5. Hermes가 만든 `receipts.json` 파일을 확인한다.
6. 채점기가 찾는 정답 키를 준비한다.
7. 채점기를 실행한다.
8. 실패하면 `receipts.json`을 고쳐 다시 채점한다.

## 1. Hermes Agent 확인

Hermes가 설치되어 있는지 확인합니다.

```powershell
hermes --version
```

도움말이 보고 싶으면 다음 명령을 실행합니다.

```powershell
hermes --help
```

## 2. Hermes 실습용 프로파일 만들기

실습 전용 프로파일을 하나 만듭니다.

프로파일 이름은 영어 소문자와 숫자만 쓰는 것이 안전합니다. 여기서는 `test1a`라고 하겠습니다.

```powershell
hermes profile create test1a --clone --description "GPT-5.6 Scaling Lab test1a receipt extraction practice"
```

이 프로파일을 기본 프로파일로 선택합니다.

```powershell
hermes profile use test1a
```

프로파일이 잘 만들어졌는지 확인합니다.

```powershell
hermes profile show test1a
```

## 3. provider와 model 설정하기

이 실습에서는 Hermes 프로파일 안에서 사용할 provider와 model을 고릅니다.

가장 쉬운 방법은 Hermes의 모델 선택 화면을 여는 것입니다.

```powershell
hermes model
```

화면이 나오면 다음을 선택합니다.

1. 사용할 provider를 고릅니다.
2. 사용할 model을 고릅니다.
3. 설정을 저장합니다.

예시는 다음과 같습니다.

| provider 예시 | model 예시 |
| --- | --- |
| `openrouter` | `openai/gpt-4.1` |
| `anthropic` | `claude-sonnet-4.5` |
| `openai` | `gpt-4.1` |
| `openai-codex` | `gpt-5.5` 또는 사용 가능한 Codex 모델 |

설정이 잘 되었는지 확인합니다.

```powershell
hermes config show
```

`Model` 항목에 provider와 model이 보이면 됩니다.

참고: provider 인증이 안 되어 있으면 Hermes가 실행 중에 401 또는 403 오류를 낼 수 있습니다. 그럴 때는 `hermes setup` 또는 `hermes auth add <provider>`로 인증을 먼저 설정합니다.

## 5. Hermes Agent에게 작업 시키기

아래 프롬프트를 그대로 복사해서 Hermes에 보냅니다.

PowerShell에서 한 번에 실행하려면 다음처럼 합니다.

```powershell
$prompt = @"
현재 폴더는 E:\ax\PRJs\gpt56-scaling 입니다.

작업 목표:
입력데이터/영수증 폴더에 있는 R01.png부터 R20.png까지 총 20장의 영수증 이미지를 읽고,
영수증 1장당 객체 1개로 정리한 JSON 배열을 만들어 주세요.

저장 위치:
runs/test1a/hermes-test/receipts.json

반드시 지켜야 할 JSON 스키마:
[
  {
    "receipt_id": "R001 형식. R001부터 R020까지 순서대로",
    "store_name": "상호명 문자열",
    "date": "YYYY-MM-DD",
    "items": [
      {
        "name": "품목명",
        "qty": 수량 정수 또는 null,
        "unit_price": 단가 정수 또는 null,
        "amount": 금액 정수 또는 null
      }
    ],
    "subtotal": 공급가액 정수 또는 null,
    "tax": 부가세 정수 또는 null,
    "total": 합계 정수 또는 null,
    "flags": [
      "문제가 있으면 한국어 문장으로 설명. 문제가 없으면 빈 배열"
    ]
  }
]

규칙:
1. 금액은 모두 원(KRW) 단위 정수로 저장해 주세요.
2. 읽을 수 없는 값은 추측하지 말고 null로 두세요.
3. null로 둔 이유나 합계 불일치 같은 문제는 flags에 적어 주세요.
4. receipts.json은 유효한 JSON이어야 합니다. Markdown 코드블록을 넣지 마세요.
5. 배열 요소는 정확히 20개여야 합니다.
6. receipt_id는 R001, R002, ..., R020 형식으로 빠짐없이 넣어 주세요.
7. 작업이 끝나면 runs/test1a/hermes-test/VERIFICATION.md 파일도 만들고, 위 규칙을 스스로 확인한 결과를 적어 주세요.
"@

hermes -z $prompt --usage-file .\runs\test1a\hermes-test\usage.json
```

Hermes를 대화형으로 쓰고 싶다면 다음처럼 실행해도 됩니다.

```powershell
hermes --tui
```

그다음 위 프롬프트를 붙여 넣으면 됩니다.

## 6. 결과 파일 확인하기

Hermes 실행이 끝나면 `receipts.json`이 생겼는지 확인합니다.

```powershell
Test-Path .\runs\test1a\hermes-test\receipts.json
```

`True`가 나오면 파일이 있는 것입니다.

JSON이 깨지지 않았는지 확인합니다.

```powershell
python -m json.tool .\runs\test1a\hermes-test\receipts.json > $null
```

아무 오류가 안 나오면 JSON 문법은 통과입니다.

배열이 20개인지 확인합니다.

```powershell
python -c "import json; data=json.load(open('runs/test1a/hermes-test/receipts.json', encoding='utf-8')); print(len(data))"
```

`20`이 나오면 됩니다.

## 7. 채점용 정답 키 준비하기

현재 채점기 `grade_test1a.py`는 정답 파일을 아래 위치에서 찾습니다.

```text
reproduction-kit/test1/receipts/ground_truth.json
```

이 파일이 없으면 채점기가 실행되지 않습니다.

먼저 정답 키를 생성합니다.

```powershell
python .\reproduction-kit\reproduction-kit\generators\gen_receipt_images.py --ground-truth-only
```

그다음 채점기가 찾는 위치로 복사합니다.

```powershell
New-Item -ItemType Directory -Force .\reproduction-kit\test1\receipts
Copy-Item .\reproduction-kit\reproduction-kit\test1\receipts\ground_truth.json .\reproduction-kit\test1\receipts\ground_truth.json -Force
```

정답 키가 준비됐는지 확인합니다.

```powershell
Test-Path .\reproduction-kit\test1\receipts\ground_truth.json
```

`True`가 나오면 됩니다.

## 8. 채점기 실행하기

이제 테스트 1A 채점기를 실행합니다.

```powershell
python .\reproduction-kit\reproduction-kit\graders\grade_test1a.py --run-dir .\runs\test1a\hermes-test
```

성공하면 JSON 형태의 결과가 출력됩니다.

중요하게 볼 항목은 다음입니다.

| 항목 | 뜻 |
| --- | --- |
| `G1_schema` | JSON 구조가 맞는지 |
| `G2_field_acc_pct` | 영수증 필드 정확도 |
| `G3_mismatch_recall` | 합계 불일치 같은 함정을 찾아냈는지 |
| `false_flags` | 잘못 표시한 문제 제기가 있는지 |
| `evaluator_pass` | 최종 통과 여부 |

`evaluator_pass`가 `true`이면 테스트 1A를 통과한 것입니다.

## 9. 실패했을 때 고치는 방법

실패해도 괜찮습니다. 아래 순서로 고치면 됩니다.

### JSON 파싱 실패

`receipts.json` 안에 Markdown 코드블록이나 설명 문장이 들어갔을 가능성이 큽니다.

해결:

- 파일 맨 앞은 `[`로 시작해야 합니다.
- 파일 맨 뒤는 `]`로 끝나야 합니다.
- ```json 같은 코드블록 표시는 제거합니다.

### 배열 20개가 아님

영수증 하나가 빠졌거나 두 번 들어갔을 수 있습니다.

해결:

- `receipt_id`가 `R001`부터 `R020`까지 있는지 확인합니다.
- 빠진 번호를 찾아 다시 Hermes에게 보완을 요청합니다.

### 정확도가 낮음

OCR을 잘못 읽은 것입니다.

해결:

- 틀린 영수증 번호를 확인합니다.
- 해당 PNG만 다시 보면서 값을 고칩니다.
- 금액은 쉼표 없이 정수로 저장합니다.

### flags가 이상함

문제가 없는 영수증에 문제를 적었거나, 문제가 있는 영수증을 놓친 것입니다.

해결:

- 합계, 공급가액, 부가세를 다시 계산합니다.
- 읽을 수 없는 값은 `null`로 두고 이유를 flags에 씁니다.
