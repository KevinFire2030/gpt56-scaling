# 테스트 3A_실습 가이드: Hermes Agent로 지구 vs 달 착지 시뮬레이터 만들기

이 문서는 Dante Labs GPT-5.6 Scaling Lab의 **테스트 3A: 달 착지 비교 시뮬레이터**를 누구나 따라 해 볼 수 있게 정리한 실습 가이드입니다.

목표는 Three.js 기반의 단일 HTML 파일 `index.html`을 만들고, 지구와 달에서 같은 힘으로 점프/낙하했을 때 실제 물리 비율이 맞는지 확인하는 것입니다. 채점기는 브라우저로 `index.html`을 열고 콘솔 로그의 낙하 시간비와 점프 높이비를 검사합니다.

중요: 이 실습은 Codex CLI가 아니라 **Hermes Agent 프로파일**로 실행합니다. 각 프로파일은 OpenRouter API 키가 아니라 기본 Codex 인증을 복사해 쓰는 `openai-codex` provider를 사용합니다.

| Sol Ultra | `sol_u` | `openai-codex` | `openai/gpt-5.6-sol` | `ultra` |
| Sol Max | `sol_max` | `openai-codex` | `openai/gpt-5.6-sol` | `max` |
| Sol High | `sol_h` | `openai-codex` | `openai/gpt-5.6-sol` | `high` |
| Luna Max | `luna_max` | `openai-codex` | `openai/gpt-5.6-luna` | `max` |

```powershell
hermes -p sol_h config set agent.reasoning_effort high
hermes -p sol_h config set delegation.reasoning_effort high
hermes -p sol_max config set agent.reasoning_effort max
hermes -p sol_max config set delegation.reasoning_effort max
hermes -p sol_u config set agent.reasoning_effort ultra

```

## 2. 실습 계획

1. 테스트 3A 채점기가 있는지 확인합니다.
2. 6개 조합별 실행 폴더를 만듭니다.
3. 각 Hermes Agent에 같은 시뮬레이터 제작 프롬프트를 넣어 `index.html`을 생성합니다.
4. 각 실행 폴더에 `VERIFICATION.md`를 남기게 합니다.
5. `grade_test3a.py`로 6개 결과물을 채점합니다.
6. `usage.json`과 실행 시간을 모아 총 토큰, 추정 비용, 점수를 표로 정리합니다.
7. 실행 중 생긴 문제와 해결 방법을 기록합니다.

## 5. 공통 프롬프트

```powershell
$basePrompt = @'
Three.js로 "지구 vs 달" 착지 비교 시뮬레이터를 만들어 주세요. 외부 파일 없이 브라우저에서 바로 열리는 단일 HTML 파일 하나(index.html)로 완성해야 합니다 (Three.js는 CDN 사용 가능).

장면 구성:
1. 화면을 좌우로 분할합니다. 왼쪽은 지구의 훈련장, 오른쪽은 달 표면입니다.
   양쪽에 동일한 우주비행사 모형이 서 있고, 오른쪽 달 장면에는 착륙선이 세워져 있습니다.
2. 착륙선 동체에는 "DANTE LABS" 로고가 선명하게 보여야 합니다.
3. 달 표면에는 크레이터가 있는 지형과 별이 보이는 검은 하늘, 지구 장면에는 하늘과 지면을 구성해 주세요. 두 세계 모두 "장난감 도형"이 아니라 실제 장소처럼 보이도록 지형·환경을 충분히 디자인해 주세요.

사용 가능 에셋:
- Three.js r128: https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js
- GLTFLoader r128: https://cdn.jsdelivr.net/gh/mrdoob/three.js@r128/examples/js/loaders/GLTFLoader.js
- 텍스처: https://cdn.jsdelivr.net/gh/mrdoob/three.js@r128/examples/textures/
- 우주비행사 3D 모델: https://modelviewer.dev/shared-assets/models/Astronaut.glb

물리 요구사항:
4. 버튼 하나로 양쪽 우주비행사가 "동시에, 같은 힘으로" 점프합니다.
   각 세계의 중력에 따라 서로 다른 높이와 체공 시간이 실제 물리대로 재현돼야 합니다.
5. 또 다른 버튼으로 양쪽 우주비행사를 같은 높이의 플랫폼에서 동시에 낙하시킵니다.
   착지 시점의 차이가 실제 물리대로 나타나야 합니다.
6. 착지 순간 발밑에서 먼지가 일어나고 지면에 발자국(눌린 자국)이 남아야 합니다.
   먼지의 움직임은 각 세계의 환경 조건에 물리적으로 맞게 표현해 주세요.

UI·계측:
7. 화면에 각 세계의 중력값, 경과 시간, 점프/낙하 버튼, 리셋 버튼을 배치해 주세요.
8. 개발자 콘솔에 실험 이벤트를 로그로 출력해 주세요:
   - 점프 시: [EARTH]/[MOON] jump_apex_m=정점높이
   - 낙하 시: [EARTH]/[MOON] fall_time_s=낙하시간
   - 착지 시: [EARTH]/[MOON] landed

카메라는 마우스로 회전·줌이 가능해야 하고, 60fps 근처를 유지해야 합니다.

[달성 기준]
1. index.html 단일 파일이 존재하고, 외부 파일 의존이 CDN 외에는 없음
2. 헤드리스 브라우저로 열어 점프·낙하 실험을 실행했을 때 콘솔 에러가 0건임을 확인
3. 요구사항 1~8이 각각 코드 어디에 구현됐는지 대응표를 VERIFICATION.md에 기록
4. 실험 로그의 실제 출력값(jump_apex_m, fall_time_s 양쪽 세계)을 VERIFICATION.md에 기록

[격리 규칙]

[완주 규칙]
```

## 6. 6개 조합 실행

```powershell
$profiles = "terra_max","sol_u","sol_max","sol_h","luna_max","sol_m"
foreach ($p in $profiles) {
  $start = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling\runs\test3a\$p"
  hermes -p $p --yolo -z $basePrompt --usage-file ".\usage.json"
  $end = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling"

  $json = [pscustomobject]@{
    profile=$p
    seconds=[math]::Round(($end-$start).TotalSeconds,1)
    html=(Test-Path ".\runs\test3a\$p\index.html")
    usage=(Test-Path ".\runs\test3a\$p\usage.json")
  } | ConvertTo-Json -Compress

  Add-Content -Encoding UTF8 -Path ".\runs\test3a\run_times.jsonl" -Value $json
  Write-Output $json
}
```
