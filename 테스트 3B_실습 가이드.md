# 테스트 3B_실습 가이드: Hermes Agent로 서울 지하철 성장 모션그래픽 만들기

이 문서는 Dante Labs GPT-5.6 Scaling Lab의 **테스트 3B: 서울 지하철 성장 모션그래픽**을 누구나 따라 할 수 있게 정리한 실습 가이드입니다.

목표는 Hermes Agent가 공개 자료를 직접 조사해서, 1974년 1호선 개통부터 2026년까지 서울 수도권 지하철 노선망이 성장하는 모습을 **15초짜리 세로형 9:16 모션그래픽**으로 만드는 것입니다.

중요: 이 실습은 Codex CLI에서 직접 제작하는 것이 아니라 **Hermes Agent 프로파일**을 호출해서 진행합니다. 각 프로파일은 OpenRouter API 키가 아니라 기본 Codex 인증을 복사해서 쓰는 `openai-codex` provider를 사용합니다.

## 1. 비교 조합

테스트 3B는 아래 6개 조합을 비교합니다.

| 조합 | Hermes 프로파일 | provider | model | reasoning |
| --- | --- | --- | --- | --- |
| Luna Max | `luna_max` | `openai-codex` | `openai/gpt-5.6-luna` | `max` |
| Sol High | `sol_h` | `openai-codex` | `openai/gpt-5.6-sol` | `high` |
| Sol Medium | `sol_m` | `openai-codex` | `openai/gpt-5.6-sol` | `medium` |
| Terra Max | `terra_max` | `openai-codex` | `openai/gpt-5.6-terra` | `max` |
| Sol Max | `sol_max` | `openai-codex` | `openai/gpt-5.6-sol` | `max` |
| Sol Ultra | `sol_u` | `openai-codex` | `openai/gpt-5.6-sol` | `ultra` |

## 2. 실습 계획

1. 테스트 3B 채점기가 있는지 확인합니다.
2. 6개 Hermes 프로파일이 준비되어 있는지 확인합니다.
3. 6개 실행 폴더를 만듭니다.
4. 공통 프롬프트를 `runs/test3b/_prompt.txt`에 저장합니다.
5. 각 프로파일을 호출해 같은 과제를 수행하게 합니다.
6. 각 결과물 폴더에서 `index.html`, `SOURCES.md`, `VERIFICATION.md`가 생성됐는지 확인합니다.
7. 공식 채점기 `grade_test3b.py`로 점수를 계산합니다.
8. `usage.json`과 실행 시간을 모아 총 토큰, 실행 시간, 추정 비용을 계산합니다.
9. 실행 중 생긴 문제와 해결 방법을 이 문서에 기록합니다.
10. 최종 결과표를 업데이트합니다.

## 6. 공통 프롬프트 저장

긴 프롬프트를 PowerShell 명령줄에 바로 붙이면 따옴표나 특수문자가 깨질 수 있습니다. 그래서 먼저 텍스트 파일로 저장하고, Hermes에는 파일 내용을 한 번에 전달합니다.

`runs/test3b/_prompt.txt`에 아래 내용을 저장합니다.

```text
서울 지하철 노선망이 개통 연도순으로 성장하는 세로형(9:16) 모션그래픽을 만들어 주세요.

1974년 1호선 개통부터 2026년까지, 도시의 지하철이 해가 갈수록 자라나는 인상을 15초 안에 전달하는 것이 목표입니다.

데이터는 직접 조사해서 확보하세요:
- 서울 수도권 지하철 각 노선(1~9호선 + 주요 광역노선)의 개통 연도, 노선 색상, 역 위치·연결 구조가 필요합니다.
- OpenStreetMap, 오픈소스 노선도 프로젝트, 위키백과 등 공개 데이터·오픈소스를 활용하세요.
- 어떤 출처를 어떻게 가공했는지 SOURCES.md에 기록하세요.
- 사실만 사용하세요. 존재하지 않는 노선·역·연도를 지어내지 마세요.

기술 스펙:
1. 최종 산출물은 브라우저에서 바로 열리는 단일 파일 index.html 하나입니다.
2. 실행 시점에 외부 리소스 요청이 없어야 합니다. 데이터·폰트·이미지 전부 파일 안에 내장하세요.
3. 1080×1920 고정 스테이지, 스크롤 없음, 오디오 없음.
4. requestAnimationFrame 기반 애니메이션으로 만드세요.
5. 시작 후 정확히 15.0초에 애니메이션이 완결되고, 이후 최종 화면을 유지해야 합니다.
6. 매 프레임 window.__mg = { t, year, lines_shown } 를 갱신하세요.

필수 화면 요소:
- 진행 연도 표시: 1974 → 2026
- 완성된 전체 노선망이 시작부터 흐린 가이드로 깔려 있어야 합니다.
- 연도에 따라 실제 노선이 그 위를 하나씩 채워져야 합니다.
- 환승역은 일반역과 시각적으로 구분되는 마커로 표시하세요.
- 주요 역 몇 곳은 개통 시 역명 라벨을 표시하세요.
- 마지막 약 1초는 완성된 전체 노선망을 유지하세요.

검증:
- index.html을 헤드리스 브라우저로 열어 콘솔 에러 0건, 런타임 외부 요청 0건임을 확인하세요.
- rAF 타임라인 15.0초 지점에서 애니메이션이 완결되고 이후 정지 화면을 유지하는지 확인하세요.
- window.__mg 계측값의 실제 시퀀스를 VERIFICATION.md에 기록하세요.
- 수집한 데이터의 출처와 가공 방법, 최종 노선 수·연도 범위를 SOURCES.md에 기록하세요.

격리 규칙:
- 웹 검색·데이터 다운로드·패키지 설치·헤드리스 브라우저는 허용됩니다.
- 단, 작업 폴더 밖의 파일·이웃 폴더·저장소의 다른 경로를 읽지 마세요.
- 이전 대화·메모리·과거 작업물을 참조하지 마세요.
- 산출물은 전부 현재 작업 폴더 안에만 생성하세요.

완주 규칙:
- 사람의 추가 입력 없이 완수하세요.
- 질문이 생기면 스스로 합리적 가정을 선택하고, 그 가정을 산출물 안에 기록하세요.
- 완료 선언 전에 달성 기준을 하나씩 스스로 실행·확인하고 VERIFICATION.md에 통과/실패로 기록하세요.
- 달성 기준이 하나라도 미충족이면 완료 선언 대신 작업을 계속하세요.
```

## 7. Hermes 실행 방법

아래 Python 래퍼를 사용하면 긴 프롬프트가 깨지지 않고, 실행 시간도 자동으로 기록됩니다.

```powershell
$env:PYTHONUTF8 = "1"
@'
import json, subprocess, time
from pathlib import Path

root = Path.cwd()
p = "luna_max"
run = root / "runs" / "test3b" / p
run.mkdir(parents=True, exist_ok=True)
prompt = (root / "runs" / "test3b" / "_prompt.txt").read_text(encoding="utf-8")

start = time.time()
rc = subprocess.run(
    ["hermes", "-p", p, "--yolo", "-z", prompt, "--usage-file", ".\\usage.json"],
    cwd=run,
).returncode

rec = {
    "profile": p,
    "seconds": round(time.time() - start, 1),
    "html": (run / "index.html").exists(),
    "sources": (run / "SOURCES.md").exists(),
    "verification": (run / "VERIFICATION.md").exists(),
    "usage": (run / "usage.json").exists(),
    "returncode": rc,
}

with (root / "runs" / "test3b" / "run_times.jsonl").open("a", encoding="utf-8") as f:
    f.write(json.dumps(rec, ensure_ascii=False) + "\n")
print(json.dumps(rec, ensure_ascii=False))
'@ | python -
```

`p` 값만 아래처럼 바꿔서 6번 실행합니다.

```text
luna_max
sol_h
sol_m
terra_max
sol_max
sol_u
```

## 8. 채점하기

```powershell
$env:PYTHONUTF8 = "1"
$profiles = "luna_max","sol_h","sol_m","terra_max","sol_max","sol_u"
foreach ($p in $profiles) {
  Write-Host "==== $p ===="
  python .\reproduction-kit\reproduction-kit\graders\grade_test3b.py --run-dir ".\runs\test3b\$p"
}
```

채점기는 대략 아래 항목을 봅니다.

| 항목 | 의미 |
| --- | --- |
| `G1_render` | `index.html` 존재, 콘솔 에러 0건, 런타임 외부 요청 0건 |
| `G2_timeline` | `window.__mg.t`가 약 15초까지 진행 |
| `G3_year` | 연도가 1974 근처에서 2026 근처까지 단조 증가 |
| `G4_growth` | 시간이 지날수록 노선 픽셀 커버리지가 증가하고 최종 색상 수가 충분함 |
| `lines_shown_max` | 최종 표시 노선 수 |
| `score_auto` | 자동 채점 점수 |
| `evaluator_pass` | 핵심 게이트 통과 여부 |

## 9. 추정 비용 계산

`usage.json`의 토큰 사용량과 Hermes 캐시에 저장된 모델 단가를 이용해 추정 비용을 계산합니다. Codex 인증 기반 실행이라 실제 청구 방식과 다를 수 있으므로, 이 값은 **OpenRouter 단가 기준 환산값**으로 봅니다.

총 토큰은 아래 항목을 더합니다.

```text
input_tokens + output_tokens + reasoning_tokens + cache_read_tokens + cache_write_tokens
```

추정 비용은 입력, 출력, 추론, 캐시 읽기/쓰기 단가를 구분해서 계산합니다.
