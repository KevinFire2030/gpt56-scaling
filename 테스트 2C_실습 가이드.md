# 테스트 2C_실습 가이드: Hermes Agent로 10분 발표 PPT 만들기

이 문서는 Dante Labs GPT-5.6 Scaling Lab의 **테스트 2C: 10분 발표 슬라이드 제작**을 누구나 따라 해 볼 수 있게 정리한 실습 가이드입니다.

목표는 `brief.md`를 읽고 10분짜리 경영진 보고용 발표 자료를 `deck.pptx`로 만드는 것입니다. 채점기는 PPT 파일을 `python-pptx`로 열어 슬라이드 수, 제목, 발표자 노트, 핵심 수치, 브랜드 컬러, 작은 폰트, 수치 모순 처리 여부를 검사합니다.

중요: 이 실습은 Codex CLI가 아니라 **Hermes Agent 프로파일**로 실행합니다. 각 프로파일은 OpenRouter API 키가 아니라 기본 Codex 인증을 복사해 쓰는 `openai-codex` provider를 사용합니다.

## 1. 비교 조합

| 조합 | Hermes 프로파일 | provider | model | reasoning |
| --- | --- | --- | --- | --- |
| Terra Max | `terra_max` | `openai-codex` | `openai/gpt-5.6-terra` | `max` |
| Sol Medium | `sol_m` | `openai-codex` | `openai/gpt-5.6-sol` | `medium` |
| Terra High | `terra_h` | `openai-codex` | `openai/gpt-5.6-terra` | `high` |
| Luna High | `luna_h` | `openai-codex` | `openai/gpt-5.6-luna` | `high` |

프로파일 확인:

```powershell

hermes profile show terra_max
hermes profile show sol_m
hermes profile show terra_h
hermes profile show luna_h
```

## 2. 실습 계획

1. 테스트 2C 채점기가 있는지 확인합니다.
2. 4개 조합별 실행 폴더를 만듭니다.
3. 같은 `brief.md`를 각 실행 폴더에 배치합니다.
4. 각 Hermes Agent에 같은 PPT 제작 프롬프트를 넣어 `deck.pptx`를 생성합니다.
5. 가능하면 `preview/` 폴더에 슬라이드 미리보기 이미지를 생성하게 합니다.
6. `grade_test2c.py`로 4개 결과물을 채점합니다.
7. `usage.json`과 실행 시간을 모아 총 토큰, 추정 비용, 점수를 표로 정리합니다.
8. 실행 중 생긴 문제와 해결 방법을 기록합니다.

## 5. brief.md 준비

원문은 `brief.md`를 투입물로 요구하지만, 현재 재현 키트 안에는 별도 `brief.md`가 포함되어 있지 않습니다. 그래서 아래처럼 채점기가 요구하는 핵심 수치와 브랜드 가이드를 포함한 재현용 브리프를 만들어 사용합니다.

```powershell
$brief = @'
# 신제품 출시 성과 보고 브리프

## 발표 목적

다음 주 경영진 회의에서 신제품 "Aurora Kit" 출시 6주 성과를 보고하고, 다음 분기 실행 결정을 요청한다.

## 청중

- CEO, CFO, CMO, 영업총괄, 제품총괄
- 숫자 중심으로 빠르게 판단하지만, 고객 반응과 실행 리스크도 함께 본다.

## 핵심 메시지

초기 수요는 예상보다 강하지만, 채널별 수익성과 재고 회전의 편차가 커서 2차 캠페인은 고효율 채널 중심으로 재배분해야 한다.

## 핵심 수치

- 출시 후 6주 누적 판매량: 12,400대
- 목표 대비 판매 초과율: 18%
- 재구매 의향 조사 응답 수: 61개 기업 고객
- 웹사이트 방문 대비 구매 전환율: 4.2%
- 반품률: 1.9%
- 다음 분기 추가 마케팅 예산 요청액: 8억 원
- 영업팀 주간 보고 기준 누적 매출: 3억 원
- 재무팀 마감 기준 누적 매출: 3.4억 원

## 수치 주의사항

영업팀 주간 보고와 재무팀 마감 보고의 누적 매출 수치가 서로 다르다. 발표에서는 두 수치를 모두 인용하되, "불일치/모순/확인 필요/검증 필요" 중 하나의 표현으로 확인 과제를 명확히 남긴다.

## 발표에 포함할 내용

- 출시 성과 요약
- 핵심 지표와 목표 대비 성과
- 채널별 시사점
- 고객 반응
- 리스크와 확인 필요 사항
- 다음 분기 의사결정 요청

## 브랜드 가이드

- 톤: 침착하고 경영진 보고에 맞는 간결한 문장
- 색상:
  - Navy: #1B2A4A
  - Amber: #F5A623
  - Paper: #FAF7F0
  - Ink: #22222A
  - Steel: #7C93B8
- 배경은 Paper 또는 White 중심으로 쓰고, 강조는 Navy와 Amber를 사용한다.
- 본문 폰트는 16pt 이상을 권장한다.
'@

$profiles = "terra_max","sol_m","terra_h","luna_h"
foreach ($p in $profiles) {
  Set-Content -Encoding UTF8 ".\runs\test2c\$p\brief.md" $brief
}
```

## 6. 공통 프롬프트

```powershell
$basePrompt = @'
다음 주 경영진 앞에서 할 신제품 출시 성과 보고 발표 자료를 만들어야 합니다.

현재 폴더의 brief.md에 발표 목적, 청중, 핵심 수치, 브랜드 가이드가 정리돼 있습니다.

brief.md를 바탕으로 10분 발표용 슬라이드를 만들어 deck.pptx 파일로 저장해 주세요.

요구사항:
1. 발표 시간은 10분입니다. 분량은 여기에 맞게 스스로 판단하세요.
2. 모든 슬라이드에 제목을 달고, 슬라이드마다 발표자 노트(말할 내용 요약)를 넣어 주세요.
3. brief.md의 브랜드 가이드(컬러, 톤)를 따라 주세요.
4. 수치를 인용할 때는 brief.md의 수치를 정확히 사용하세요.
5. 표지 → 개요 → 본론 → 마무리의 완결된 구조로 만들어 주세요.

[달성 기준]
1. deck.pptx 파일이 존재하고 python-pptx로 정상 파싱됨
2. 모든 슬라이드에 제목과 발표자 노트가 존재
3. 표지·마무리 슬라이드 포함 완결 구조
4. 슬라이드를 이미지로 렌더(또는 스크린샷)해 preview/ 폴더에 저장

[완주 규칙]
- 이 목표는 사람의 추가 입력 없이 완수해야 합니다. 질문이 생기면 스스로 합리적 가정을 선택하고, 그 가정을 산출물 안에 기록하세요.
- 완료를 선언하기 전에 위 달성 기준을 하나씩 스스로 실행·확인하고, 체크 결과를 VERIFICATION.md 파일에 기준별 통과/실패로 기록하세요.
- 달성 기준이 하나라도 미충족이면 완료 선언 대신 작업을 계속하세요.
'@
```

## 7. 4개 조합 실행

```powershell
$profiles = "terra_max","sol_m","terra_h","luna_h"
foreach ($p in $profiles) {
  $start = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling\runs\test2c\$p"
  hermes -p $p --yolo -z $basePrompt --usage-file ".\usage.json"
  $end = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling"

  $json = [pscustomobject]@{
    profile=$p
    seconds=[math]::Round(($end-$start).TotalSeconds,1)
    deck=(Test-Path ".\runs\test2c\$p\deck.pptx")
    usage=(Test-Path ".\runs\test2c\$p\usage.json")
  } | ConvertTo-Json -Compress

  Add-Content -Encoding UTF8 -Path ".\runs\test2c\run_times.jsonl" -Value $json
  Write-Output $json
}
```
