# 테스트 2B_실습 가이드: Hermes Agent로 Apple KR 사이트 클론하기

이 문서는 Dante Labs GPT-5.6 Scaling Lab의 **테스트 2B: 사이트 클론 재현 충실도**를 누구나 따라 해 볼 수 있게 정리한 실습 가이드입니다.

목표는 `https://www.apple.com/kr/` 첫 화면을 참고해 각 Hermes 프로파일이 `index.html` 하나를 만들고, 채점기로 DOM/콘솔/SSIM/모바일 가로 스크롤을 확인하는 것입니다.

중요: 이 실습은 Codex CLI가 아니라 **Hermes Agent 프로파일**로 실행합니다. 각 프로파일은 OpenRouter API 키가 아니라 기본 Codex 인증을 복사해 쓰는 `openai-codex` provider를 사용했습니다.

## 4. 원본 Apple KR 스냅샷 준비

채점기는 아래 파일을 원본 비교 이미지로 사용합니다.

```text
reproduction-kit/test2b/_original_apple/screenshot_1440_fold.png
```

이번 실측에서는 Node.js의 `playwright` 패키지가 없어서 Python Playwright로 캡처했습니다.

```powershell
New-Item -ItemType Directory -Force .\reproduction-kit\test2b\_original_apple

$code = @'
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(channel="chrome", headless=True)
    page = browser.new_page(viewport={"width": 1440, "height": 900})
    page.goto("https://www.apple.com/kr/", wait_until="networkidle", timeout=60000)
    page.wait_for_timeout(2500)
    page.screenshot(path="reproduction-kit/test2b/_original_apple/screenshot_1440_fold.png")
    browser.close()
'@

$code | python -
Test-Path .\reproduction-kit\test2b\_original_apple\screenshot_1440_fold.png
```

주의: Apple KR 페이지는 시간이 지나면 바뀝니다. 따라서 이 실습의 SSIM은 Dante 원문 캡처가 아니라 **실행 시점에 새로 캡처한 현재 Apple KR 화면** 기준입니다.

## 7. 5개 조합 실행

프로파일별로 폴더를 옮겨 실행합니다.

```powershell
$profiles = "sol_m","luna_max","terra_m","terra_h","terra_max"
foreach ($p in $profiles) {
  $start = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling\runs\test2b\$p"
  hermes -p $p --yolo -z $basePrompt --usage-file ".\usage.json"
  $end = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling"

  $json = [pscustomobject]@{
    profile=$p
    seconds=[math]::Round(($end-$start).TotalSeconds,1)
    html=(Test-Path ".\runs\test2b\$p\index.html")
    usage=(Test-Path ".\runs\test2b\$p\usage.json")
  } | ConvertTo-Json -Compress

  Add-Content -Encoding UTF8 -Path ".\runs\test2b\run_times.jsonl" -Value $json
  Write-Output $json
}
```

## 9. 실제 테스트 과정에서 나온 문제와 해결 방법

| 문제 | 원인 | 해결 방법 |
| --- | --- | --- |
| Node.js에서 `require("playwright")` 실패 | 현재 Node 런타임에 Playwright 패키지가 없었음 | Python Playwright는 설치되어 있어 `$code | python -` 방식으로 원본 스냅샷 캡처 |
| PowerShell `python -c`로 `with sync_playwright()` 실행 시 `SyntaxError` | 여러 줄 `with` 블록을 한 줄 명령으로 넘기기 어려움 | PowerShell here-string을 만든 뒤 표준입력으로 Python에 전달 |
| 일부 실행에서 빈 `({src` 파일 생성 | 모델이 Apple 이미지/소스 수집 중 잘못 해석된 문자열을 파일명으로 만든 것으로 추정 | 채점 대상은 `index.html`이므로 영향 없음. 재현 기록에는 남기고, 필요하면 실행 폴더 정리 |
| `sol_m` 실행 후 결과는 완료됐지만 터미널이 오래 붙잡힘 | Hermes 마무리 출력/검증 후 세션 종료가 지연됨 | `usage.json`의 `completed: true`와 `VERIFICATION.md` 생성 확인 후 세션 정리 |
| `VERIFICATION.md` 한글이 터미널에서 깨져 보임 | PowerShell 출력 인코딩 문제 | 파일 저장은 유지하고, 채점/결과표는 UTF-8 Python 실행으로 처리 |
| OpenRouter 비용 계산 시 모델 ID 매칭 실패 | Hermes usage의 model은 `gpt-5.6-sol`처럼 provider 접두사가 빠져 있음 | `gpt-5.6-sol` -> `openai/gpt-5.6-sol`처럼 매핑해서 메타데이터 가격 적용 |
