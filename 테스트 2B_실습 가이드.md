# 테스트 2B 실습 가이드: Hermes Agent로 Apple KR 사이트 클로닝하기

이 문서는 **테스트 2B: 사이트 클로닝 재현 충실도**를 Hermes Agent로 따라 하는 실습 가이드입니다.

목표는 `https://www.apple.com/kr/` 첫 화면을 시각적으로 최대한 비슷하게 복제해 `index.html` 단일 파일로 저장하는 것입니다.

중요: 실습은 Codex CLI에서 직접 하는 것이 아니라 **Hermes Agent 프로파일**로 실행합니다.

## 목표

입력:

```text
https://www.apple.com/kr/
```

출력:

```text
runs/test2b/sol_m/index.html
runs/test2b/luna_max/index.html
runs/test2b/terra_m/index.html
runs/test2b/terra_h/index.html
runs/test2b/terra_max/index.html
```

채점기는 각 실행 폴더에서 아래 파일을 찾습니다.

```text
index.html
```

## 비교 조합

단테랩스 원문 기준 테스트 2B 비교 조합은 아래 5개입니다.

| 조합 | 프로파일 | model | reasoning |
| --- | --- | --- | --- |
| Sol Medium | `sol_m` | `openai/gpt-5.6-sol` | `medium` |
| Luna Max | `luna_max` | `openai/gpt-5.6-luna` | `max` |
| Terra Medium | `terra_m` | `openai/gpt-5.6-terra` | `medium` |
| Terra High | `terra_h` | `openai/gpt-5.6-terra` | `high` |
| Terra Max | `terra_max` | `openai/gpt-5.6-terra` | `max` |

## 전체 계획

1. 5개 Hermes 프로파일을 확인합니다.
2. 각 프로파일이 기본 Codex 인증을 재사용하도록 설정합니다.
3. 조합별 실행 폴더를 만듭니다.
4. Apple KR 원본 첫 화면 스냅샷을 채점기 위치에 준비합니다.
5. 각 Hermes Agent로 `index.html` 단일 파일을 생성합니다.
6. Playwright 기반 채점기를 실행합니다.
7. 사용 토큰, 실행시간, 추정 비용을 모아 최종 결과표를 작성합니다.
8. 테스트 중 나온 문제와 해결 방법을 기록합니다.

## 1. 테스트 2B가 하는 일 이해하기

이 테스트는 데이터를 읽는 문제가 아니라 **웹페이지를 눈으로 비슷하게 재현하는 문제**입니다.

에이전트가 해야 할 일:

1. Apple KR 홈페이지를 참고합니다.
2. 헤더, 내비게이션, 히어로, 주요 제품 섹션을 재현합니다.
3. `index.html` 하나만 만듭니다.
4. 1440px 데스크톱 화면에서 원본과 비슷해야 합니다.
5. 375px 모바일 화면에서 가로 스크롤이 없어야 합니다.
6. 콘솔 에러가 없어야 합니다.

## 5. 원본 스냅샷 준비하기

채점기 `grade_test2b.py`는 원본 첫 화면 스냅샷을 아래 위치에서 찾습니다.

```text
reproduction-kit/test2b/_original_apple/screenshot_1440_fold.png
```

아래 PowerShell 블록으로 현재 Apple KR 첫 화면을 캡처합니다.

```powershell
New-Item -ItemType Directory -Force .\reproduction-kit\test2b\_original_apple

$script = @'
const { chromium } = require("playwright");
(async () => {
  const browser = await chromium.launch({ channel: "chrome", headless: true });
  const page = await browser.newPage({ viewport: { width: 1440, height: 900 } });
  await page.goto("https://www.apple.com/kr/", { waitUntil: "networkidle", timeout: 60000 });
  await page.waitForTimeout(2500);
  await page.screenshot({ path: "reproduction-kit/test2b/_original_apple/screenshot_1440_fold.png" });
  await browser.close();
})();
'@

Set-Content -Encoding UTF8 .\capture_apple_snapshot.js $script
node .\capture_apple_snapshot.js
Remove-Item .\capture_apple_snapshot.js
```

확인합니다.

```powershell
Test-Path .\reproduction-kit\test2b\_original_apple\screenshot_1440_fold.png
```

## 6. 공통 프롬프트 준비

```powershell
$basePrompt = @'
작업을 지금 실제로 수행하세요. 설명만 하지 마세요.

https://www.apple.com/kr/ 홈페이지를 시각적으로 최대한 동일하게 복제해 주세요.

요구사항:
1. 산출물은 단일 HTML 파일 index.html 하나입니다.
2. 외부 의존은 CDN 폰트/아이콘 정도만 허용합니다.
3. 기준 뷰포트는 데스크톱 1440px입니다.
4. 375px 모바일 뷰포트에서도 가로 스크롤 없이 정상 표시돼야 합니다.
5. 헤더 내비게이션, 히어로 섹션, 본문 주요 섹션, 푸터의 구조와 문구를 재현하세요.
6. 첫 방문 시 뜨는 공지 팝업이나 모달은 닫힌 상태 기준으로 제외합니다.
7. 채점 텍스트인 MacBook Air, iPhone, 스토어, iPad, Watch, AirPods, 고객지원, 구입하기가 페이지 본문에 자연스럽게 들어가야 합니다.

달성 기준:
1. index.html 단일 파일이 존재
2. 헤드리스 브라우저로 열어 콘솔 에러 0건 확인
3. 1440px 스크린샷을 캡처해 원본과 비교한 메모를 VERIFICATION.md에 기록

완주 규칙:
- 이 목표는 사람의 추가 입력 없이 완수해야 합니다.
- 질문이 생기면 스스로 합리적 가정을 선택하고, 그 가정을 산출물 안에 기록하세요.
- 완료를 선언하기 전에 위 달성 기준을 하나씩 스스로 실행/확인하고, 체크 결과를 VERIFICATION.md 파일에 기준별 통과/실패로 기록하세요.
- 달성 기준이 하나라도 미충족이면 완료 선언 대신 작업을 계속하세요.
'@
```

## 7. 순차 실행하기

```powershell
$profiles = "sol_m","luna_max","terra_m","terra_h","terra_max"
foreach ($p in $profiles) {
  $start = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling\runs\test2b\$p"
  hermes -p $p --yolo -z $basePrompt --usage-file ".\usage.json"
  $end = Get-Date
  Set-Location "E:\ax\PRJs\gpt56-scaling"
  [pscustomobject]@{
    profile=$p
    seconds=[math]::Round(($end-$start).TotalSeconds,1)
    html=(Test-Path ".\runs\test2b\$p\index.html")
    usage=(Test-Path ".\runs\test2b\$p\usage.json")
  } | ConvertTo-Json -Compress | Tee-Object -FilePath ".\runs\test2b\run_times.jsonl" -Append
}
```

## 10. 실패했을 때 고치는 방법

### 원본 스냅샷 없음

- `reproduction-kit/test2b/_original_apple/screenshot_1440_fold.png`가 없으면 채점기가 실패합니다.
- 5번의 원본 스냅샷 캡처 단계를 먼저 실행합니다.

### Playwright 또는 Chrome 오류

- 채점기는 Playwright와 Chrome 채널을 사용합니다.
- Chrome이 없으면 설치하거나 채점기에서 `channel="chrome"` 부분을 현재 환경에 맞게 수정해야 합니다.

### index.html 없음

- Hermes가 채팅 답변만 하고 파일을 저장하지 않은 경우입니다.
- 프롬프트 첫 줄에 “작업을 지금 실제로 수행하세요. 설명만 하지 마세요.”를 넣습니다.
- `.bat` alias 대신 `hermes -p <profile> --yolo -z ...`로 실행합니다.

### DOM 점수가 낮음

- `MacBook Air`, `iPhone`, `스토어`, `iPad`, `Watch`, `AirPods`, `고객지원`, `구입하기` 텍스트가 본문에 들어갔는지 확인합니다.
- 숨김 텍스트로 넣기보다 실제 내비게이션과 섹션 문구에 자연스럽게 넣는 것이 좋습니다.

### SSIM이 낮음

- 1440px 첫 화면의 배치, 배경색, 큰 제품명, 헤더 높이, 섹션 간 여백을 원본과 맞춥니다.
- 어두운 히어로/밝은 히어로 순서가 원본과 다르면 점수가 크게 떨어질 수 있습니다.
- 원본 스냅샷을 캡처한 시점의 Apple 페이지가 바뀌면 결과도 달라질 수 있습니다.

### 375px 모바일 가로 스크롤 발생

- 고정 폭 요소를 줄입니다.
- `body { overflow-x: hidden; }`를 추가합니다.
- 제품 카드나 이미지에 `max-width: 100%`를 지정합니다.
