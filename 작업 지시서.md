# 작업 지시서: Ollama gemma4로 7개 벤치마크 재실행

이 문서는 회사 PC에서 Codex CLI에게 시킬 작업 지시서입니다.

회사 PC 조건:

- Windows에 Codex CLI 설치됨
- WSL에 Hermes Agent 설치됨
- 회사 PC 또는 사내망의 Ollama 서버에서 `gemma4` 모델 제공 중
- Ollama 서버 주소: `http://10.252.87.46:11434`

## 1. 목적

이 저장소의 기존 7개 벤치마크 테스트를 GPT-5.6 계열이 아니라 **Ollama에서 서빙 중인 gemma4 모델 하나**로 다시 수행합니다.

목적은 다음과 같습니다.

1. Hermes Agent에서 `gemma4` 전용 프로파일을 만든다.
2. 기존 7개 테스트를 같은 절차로 gemma4에 실행시킨다.
3. 테스트별 산출물, 채점 결과, 문제와 해결 방법을 기록한다.
4. 새 문서 `테스트 4A_실습가이드.md`에 gemma4 단일 모델 재현 가이드를 만든다.
5. 최종 결과를 GitHub 원격 저장소에 푸쉬한다.

### 3-2. WSL/Hermes/Ollama 연결 확인

회사 PC에서는 Hermes Agent가 WSL에 설치되어 있으므로, Codex CLI에게 아래 작업은 WSL 셸에서 실행하게 합니다.

PowerShell에서 WSL로 들어갑니다.

```powershell

```

WSL 안에서 저장소로 이동합니다. Windows 경로 `D:\AX\PRJs\github\gpt56-scaling`은 보통 WSL에서 `/mnt/d/AX/PRJs/github/gpt56-scaling`로 보입니다.

```bash
cd /mnt/d/AX/PRJs/github/gpt56-scaling
pwd
```

Hermes가 설치되어 있는지 확인합니다.

```bash
hermes --version
hermes profile list
```

Ollama 서버가 WSL에서 접근 가능한지 확인합니다.

```bash
curl http://10.252.87.46:11434/api/tags
```

응답에 `gemma4`가 보이면 정상입니다. 안 보이면 아래도 확인합니다.

```bash
curl http://10.252.87.46:11434/api/version
```

Ollama OpenAI 호환 API도 확인합니다.

```bash
curl http://10.252.87.46:11434/v1/models
```

주의:

- Hermes가 native Ollama provider를 지원하면 base URL은 `http://10.252.87.46:11434`를 사용합니다.
- Hermes가 OpenAI-compatible provider로 연결해야 하면 base URL은 `http://10.252.87.46:11434/v1`를 사용합니다.
- 모델명은 실제 Ollama tags 응답의 이름을 우선합니다. 사용자가 지정한 기본 모델명은 `gemma4`입니다.

### 3-3. Hermes Agent 프로파일 생성

이번 테스트에서는 프로파일을 하나만 만듭니다.

| 항목 | 값 |
| --- | --- |
| 프로파일 이름 | `gemma4` |
| LLM | `gemma4` |
| 서버 | `http://10.252.87.46:11434` |
| 실행 방식 | Hermes Agent |
| 테스트 반복 | 7개 테스트를 같은 프로파일로 순차 실행 |

먼저 기본 프로파일을 복제합니다.

```bash
hermes profile create gemma4 --clone --description "Test4A Ollama gemma4 benchmark agent"
```

만약 `--clone` 옵션이 동작하지 않으면 현재 기본 프로파일 이름을 확인한 뒤 복제합니다.

```bash
hermes profile list
hermes profile create gemma4 --clone-from default --description "Test4A Ollama gemma4 benchmark agent"
```

그 다음 모델 설정을 합니다. 가장 안전한 방법은 Hermes의 모델 선택 화면을 여는 것입니다.

```bash
hermes -p gemma4 model
```

화면에서 아래 중 가능한 방식을 선택합니다.

방법 A: native Ollama provider가 있는 경우

```text
provider: ollama
base_url: http://10.252.87.46:11434
model: gemma4
```

방법 B: OpenAI-compatible provider로 연결하는 경우

```text
provider: openai-compatible
base_url: http://10.252.87.46:11434/v1
model: gemma4
api_key: ollama
```

설정 후 간단한 호출 테스트를 합니다.

```bash
hermes -p gemma4 --yolo -z "한 문장으로 자기소개를 해줘. 그리고 현재 사용 모델명을 말해줘."
```

정상 응답이 오면 프로파일 준비 완료입니다.

### 3-7. Hermes 실행 래퍼

긴 한글 프롬프트를 직접 `-z`에 넣으면 터미널 인자 처리가 깨질 수 있습니다. Python 래퍼를 사용합니다.

`runs/test4a/run_one.py`를 만듭니다.

```python
import json
import subprocess
import sys
import time
from pathlib import Path


def main() -> int:
    if len(sys.argv) != 4:
        print("usage: python runs/test4a/run_one.py <test_id> <run_dir> <prompt_file>")
        return 2

    test_id = sys.argv[1]
    run_dir = Path(sys.argv[2]).resolve()
    prompt_file = Path(sys.argv[3]).resolve()
    run_dir.mkdir(parents=True, exist_ok=True)
    prompt = prompt_file.read_text(encoding="utf-8")

    start = time.time()
    rc = subprocess.run(
        ["hermes", "-p", "gemma4", "--yolo", "-z", prompt, "--usage-file", "./usage.json"],
        cwd=run_dir,
    ).returncode

    rec = {
        "test": test_id,
        "profile": "gemma4",
        "seconds": round(time.time() - start, 1),
        "returncode": rc,
        "usage": (run_dir / "usage.json").exists(),
        "finished_at": time.strftime("%Y-%m-%d %H:%M:%S"),
    }

    log_dir = Path.cwd() / "runs" / "test4a"
    log_dir.mkdir(parents=True, exist_ok=True)
    with (log_dir / "run_times.jsonl").open("a", encoding="utf-8") as f:
        f.write(json.dumps(rec, ensure_ascii=False) + "\n")
    print(json.dumps(rec, ensure_ascii=False))
    return rc


if __name__ == "__main__":
    raise SystemExit(main())
```

실행 예시는 아래와 같습니다.

```bash
python runs/test4a/run_one.py test1a runs/test1a/gemma4 runs/test4a/prompts/test1a_prompt.txt
```

### 3-8. 실습가이드를 순차적으로 읽어서 테스트 실행

Codex CLI는 반드시 아래 순서로 기존 실습 가이드를 읽고 실행합니다.

1. `테스트 1A_실습 가이드.md`를 읽는다.
2. gemma4용 프롬프트를 만든다.
3. `runs/test1a/gemma4`에서 Hermes를 실행한다.
4. 채점기를 실행한다.
5. 문제와 해결 방법을 `테스트 4A_실습가이드.md`에 기록한다.
6. 결과표를 업데이트한다.
7. 다음 테스트로 넘어간다.

위 과정을 1A, 1B, 2A, 2B, 2C, 3A, 3B 순서로 반복합니다.

- Ollama 모델은 context 길이와 도구 사용 능력이 GPT-5.6 계열과 다를 수 있으므로, 실패해도 임의로 결과를 조작하지 말고 그대로 기록합니다.
