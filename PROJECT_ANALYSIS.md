# GPT-5.6 Scaling Lab 프로젝트 분석

## 한 줄 요약

이 저장소는 단테랩스의 글 **"GPT-5.6 모델x추론 스케일링 실전 테스트"**를 로컬에서 보존하고, 해당 글에서 공개한 벤치마크 입력 데이터와 재현 키트를 함께 묶은 프로젝트입니다. 핵심 목적은 여러 모델/추론 설정이 같은 과제를 얼마나 정확하고 비용 효율적으로 수행하는지 재현하거나 검증하는 것입니다.

## 프로젝트 성격

이 폴더는 일반적인 애플리케이션 소스 저장소라기보다 **벤치마크 재현 자료 저장소**에 가깝습니다.

- 공개 페이지의 HTML/PDF 보존본
- 벤치마크에 투입되는 입력 데이터
- 입력 데이터를 생성하는 스크립트
- 모델 산출물을 평가하는 채점기
- 과제 지시문과 평가 설계 자료

즉, "모델에게 과제를 던지고, 산출물을 파일로 받은 뒤, 채점기로 합격 여부와 품질을 측정하는" 실험 환경입니다.

## 주요 구성

| 경로 | 역할 |
| --- | --- |
| `dante-labs-gpt56-scaling-lab.html` | 단테랩스 원문 페이지의 HTML 저장본 |
| `dante-labs-gpt56-scaling-lab.pdf` | 같은 페이지의 PDF 보존본 |
| `prompt.md` | 벤치마크에 사용된 과제 지시문 전문 |
| `입력데이터.zip` | 공개 입력 데이터 원본 ZIP |
| `입력데이터/` | 압축 해제된 실제 테스트 입력 데이터 |
| `reproduction-kit.zip` | 재현 키트 원본 ZIP |
| `reproduction-kit/reproduction-kit/` | 압축 해제된 생성기와 채점기 코드 |

현재 Git 기준으로는 총 42개 파일이 추적되고 있습니다.

## 입력 데이터

`입력데이터/` 폴더에는 3종류의 실무형 입력 자료가 들어 있습니다.

| 파일/폴더 | 내용 |
| --- | --- |
| `입력데이터/영수증/R01.png` ~ `R20.png` | 영수증 이미지 20장. OCR/구조화 추출 과제용 |
| `입력데이터/함정매출_sales.csv` | 13,091행 x 6컬럼의 일별 매출 데이터. 컬럼은 `date`, `store`, `revenue`, `transactions`, `promo`, `avg_temp_c` |
| `입력데이터/원장_ledger.xlsx` | 500행 거래 원장. 거래처/항목/수량/단가/금액 기반 정산 과제용 |
| `입력데이터/정산서_양식.xlsx` | 원장 데이터를 채워 넣을 정산서 템플릿 |

데이터는 글에서도 명시하듯 실제 상호나 인물과 무관한 합성 데이터입니다.

## 재현 키트

`reproduction-kit/reproduction-kit/` 아래는 크게 두 묶음입니다.

### `generators/`

입력 데이터와 실험 실행 보조 자료를 만드는 코드입니다.

| 파일 | 역할 |
| --- | --- |
| `gen_receipt_images.py` | 영수증 PNG와 정답 JSON 생성 |
| `gen_ledger_xlsx.py` | 거래 원장, 정산서 양식, 정답 엑셀 생성 |
| `gen_sales_csv.py` | 함정이 포함된 매출 CSV 생성 |
| `run_router.py` | 난이도 분류 및 자동 승급 흐름 실험용 러너 |
| `collect_metrics.py` | Codex 세션 로그에서 시간/토큰/비용 지표 수집 |

### `graders/`

모델 산출물을 자동 또는 반자동으로 평가하는 채점기입니다.

| 파일 | 평가 대상 |
| --- | --- |
| `grade_test1a.py` | 영수증 이미지 20장의 구조화 JSON 추출 |
| `grade_test1b.py` | 원장 기반 정산서 완성 엑셀 |
| `grade_test2a.py` | 함정 매출 데이터 분석 보고서와 분석 코드 |
| `grade_test2b.py` | Apple Korea 사이트 클론 HTML의 시각적 재현도 |
| `grade_test2c.py` | 10분 발표용 PPT 작성 결과 |
| `grade_test3a.py` | Three.js 기반 지구 vs 달 착지 물리 시뮬레이션 |
| `grade_test3b.py` | 서울 지하철 성장 모션그래픽 HTML |
| `common.py` | JSON 결과 출력, 이미지 SSIM 비교 등 공통 유틸리티 |

채점 방식은 JSON 스키마, 엑셀 구조, 키워드 리콜, Playwright 렌더링, SSIM, 콘솔 로그, `window.__mg` 계측값 등 과제별로 다릅니다. 일부 평가는 자동 점수 뒤에 사람이 최종 확인하도록 설계되어 있습니다.

## 벤치마크 과제 흐름

이 프로젝트가 가정하는 흐름은 다음과 같습니다.

1. 모델/추론 설정 조합을 정한다.
2. `prompt.md`에 있는 동일 지시문과 `입력데이터/` 자료를 모델에게 제공한다.
3. 모델이 요구된 산출물을 파일로 만든다.
4. `graders/`의 해당 채점기를 실행해 통과 여부와 점수를 확인한다.
5. 시간, 비용, 품질을 비교한다.

과제는 단순 텍스트 답변이 아니라 파일 산출물 중심입니다. 예를 들어 `receipts.json`, `정산서_완성.xlsx`, `report.md`, `analysis.py`, `index.html`, `deck.pptx`, `VERIFICATION.md`, `SOURCES.md` 같은 결과물이 요구됩니다.

## 왜 중요한 프로젝트인가

이 저장소의 가치는 "모델이 좋아 보인다"는 주관적 평가가 아니라, 같은 과제를 반복 가능한 자료와 채점 코드로 비교하려는 데 있습니다.

- 추론 effort를 올렸을 때 실제 품질이 좋아지는지 확인 가능
- 비싼 조합과 저렴한 조합의 비용 대비 성능 비교 가능
- OCR, 엑셀 자동화, 데이터 분석, 웹 클론, PPT 생성, 3D 시뮬레이션, 데이터 수집형 모션그래픽 등 서로 다른 업무 유형을 한 번에 비교 가능
- 모델이 지시문 속 함정, 형식 제약, 검증 루프를 얼마나 잘 다루는지 관찰 가능

## 현재 저장소 상태에서의 주의점

- `prompt.md`와 일부 `README.md`는 Windows 콘솔에서 인코딩이 깨져 보일 수 있습니다. 파일 자체는 벤치마크 자료로 보존되어 있으므로, UTF-8을 올바르게 처리하는 편집기에서 확인하는 편이 좋습니다.
- `reproduction-kit/` 안에는 채점기와 생성기 중심의 코드가 있으며, 완성된 모델별 실험 산출물 전체가 들어 있는 구조는 아닙니다.
- Playwright 기반 채점기(`grade_test2b.py`, `grade_test3a.py`, `grade_test3b.py`)는 실행 환경에 브라우저/의존성이 필요할 수 있습니다.
- `입력데이터.zip`은 한글 파일명 인코딩 문제 때문에 Windows 내장 `Expand-Archive`에서 실패할 수 있습니다. 이 작업공간에서는 UTF-8 파일명을 복원해 `입력데이터/`로 정상 추출했습니다.

이 저장소를 활용하면 새 모델이나 새 추론 설정이 나왔을 때 같은 과제를 다시 돌려 보고, 기존 결과와 비용 대비 품질을 비교할 수 있습니다.
