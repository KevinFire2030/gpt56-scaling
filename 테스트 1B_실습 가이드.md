# 테스트 1B 실습 가이드: Hermes Agent로 월별 정산서 채우기

이 문서는 **테스트 1B: 월별 정산서 폼 채우기**를 Hermes Agent로 따라 하는 실습 가이드입니다.

핵심은 간단합니다.

- `ledger.xlsx` 원장 데이터를 읽습니다.
- `정산서_양식.xlsx`의 노란 칸을 채웁니다.
- 완성본을 `정산서_완성.xlsx`로 저장합니다.
- 채점기로 36개 셀의 정답률을 확인합니다.

중요: 실습은 Codex CLI에서 직접 하는 것이 아니라 **Hermes Agent 프로파일**로 실행합니다.

## 목표

입력:

```text
입력데이터/원장_ledger.xlsx
입력데이터/정산서_양식.xlsx
```

출력:

```text
runs/test1b/luna_m/정산서_완성.xlsx
runs/test1b/luna_h/정산서_완성.xlsx
runs/test1b/terra_m/정산서_완성.xlsx
runs/test1b/sol_m/정산서_완성.xlsx
```

최종 비교 대상:

| 조합 | 프로파일 | 결과 폴더 |
| --- | --- | --- |
| Luna Medium | `luna_m` | `runs/test1b/luna_m/` |
| Luna High | `luna_h` | `runs/test1b/luna_h/` |
| Terra Medium | `terra_m` | `runs/test1b/terra_m/` |
| Sol Medium | `sol_m` | `runs/test1b/sol_m/` |

## 전체 계획

1. 프로젝트 폴더로 이동합니다.
2. 입력 엑셀 파일 2개가 있는지 확인합니다.
3. Hermes 프로파일 4개가 준비되어 있는지 확인합니다.
4. 각 프로파일이 Codex 인증을 재사용하도록 맞춥니다.
5. 조합별 실행 폴더를 만들고 입력 엑셀을 복사합니다.
6. 각 Hermes 프로파일로 정산서 완성 작업을 실행합니다.
7. `정산서_완성.xlsx`가 생겼는지 확인합니다.
8. 정답 키를 준비합니다.
9. 채점기를 실행합니다.
10. 최종 결과 표를 작성합니다.

## 1. 테스트 1B가 하는 일 이해하기

`원장_ledger.xlsx`에는 거래 내역이 들어 있습니다.

대략 이런 열이 있습니다.

| 열 | 뜻 |
| --- | --- |
| 일자 | 거래 날짜 |
| 거래처 | 업체 이름 |
| 항목 | 산 물건 |
| 수량 | 몇 개 샀는지 |
| 단가 | 1개 가격 |
| 금액 | 수량 x 단가 |

`정산서_양식.xlsx`에는 노란 칸이 있습니다.

Hermes Agent가 해야 할 일은 다음입니다.

1. 원장을 읽습니다.
2. 거래처별로 4월, 5월, 6월 금액을 더합니다.
3. 정산서 양식의 노란 칸에 값을 채웁니다.
4. 양식 구조는 바꾸지 않습니다.
5. 완성 파일을 `정산서_완성.xlsx`로 저장합니다.

주의: 양식 안에는 작은 글씨 안내 문구가 있습니다. 이 문구를 읽고 따라야 좋은 점수가 나옵니다.

## 2. Hermes 프로파일 확인

1A 실습을 이미 했다면 아래 프로파일은 이미 있을 가능성이 큽니다.

```powershell

```

필요한 프로파일:

```text

luna_h
terra_m
sol_m
```

```powershell

hermes profile create terra_m --clone --description "Test1B Terra Medium benchmark agent"

```

$jobs += Start-Job -Name terra_m -ScriptBlock {

## 12. 실패했을 때 고치는 방법

### Hermes가 확인 답변만 하고 파일을 만들지 않음

- `.bat` alias가 긴 멀티라인 프롬프트를 제대로 전달하지 못한 경우일 수 있습니다.
- `luna_m -z ...` 대신 `hermes -p luna_m --yolo -z ...`처럼 직접 프로파일을 지정해 실행합니다.
- 이번 실측도 이 방식으로 실행했습니다.

### `정산서_완성.xlsx 없음`

- Hermes가 결과 파일을 다른 이름으로 저장했는지 확인합니다.
- 파일명을 반드시 `정산서_완성.xlsx`로 바꿉니다.
- 실행 폴더 안에 저장되어 있어야 합니다.

### 열기 실패 또는 시트명 변경

- 시트 이름은 `월별정산서`여야 합니다.
- 새 엑셀 파일을 처음부터 만들기보다 `정산서_양식.xlsx`를 복사해서 채우는 방식이 안전합니다.

### 구조 불변 실패

- 헤더 행, 거래처 순서, 합계 행을 바꾸면 안 됩니다.
- 노란 칸 외 셀은 건드리지 않는 것이 좋습니다.

### 셀 정확도 낮음

- 월별 집계 범위가 2026년 4월, 5월, 6월인지 확인합니다.
- 거래처별 합계와 전체 합계를 다시 계산합니다.
- 양식의 작은 글씨 안내 문구를 다시 확인합니다.
- 취소 거래처럼 보이는 음수 금액을 어떻게 처리했는지 확인합니다.
- 금액 절사 규칙을 적용했는지 확인합니다.

## 14. 최종 결과물 표

5개가 아니라 4개 조합을 실행합니다.

아래 표는 이번 로컬 실습에서 Hermes Agent와 Codex 인증으로 직접 실행하고 채점한 결과입니다.

| 조합 | 판정 | 점수 | 총 토큰 | 실행시간 | 추정 비용 |
| --- | --- | ---: | ---: | ---: | ---: |
| Luna Medium | ✅ | 100.0 | 383,967 | 2분 36초 | $0.2255 |
| Luna High | ✅ | 100.0 | 347,379 | 1분 49초 | $0.2005 |
| Terra Medium | ✅ | 100.0 | 237,302 | 1분 23초 | $0.2111 |
| Sol Medium | ✅ | 100.0 | 454,757 | 2분 54초 | $1.3169 |

실측 요약:

- 4개 조합 모두 `G1_structure=true`, `G2_filled=36/36`, `G3_cell_acc_pct=100.0`, `G4_rule_diagnosis=OK`로 통과했습니다.
- 총 토큰은 `input + output + reasoning + cache_read + cache_write`로 계산했습니다.
- 추정 비용은 Codex 인증의 실제 결제 비용이 아니라 OpenRouter 공개 단가 기준의 비교용 계산값입니다.
- Codex 인증 실행 로그의 `estimated_cost_usd`는 구독 포함 실행이라 `0.0`으로 기록됩니다.

단테랩스 원문 기준 결과는 아래와 같습니다.

| 조합 | 점수 | 비용 |
| --- | ---: | ---: |
| Luna Medium | 100 | $0.199 |
| Luna High | 100 | $0.348 |
| Terra Medium | 100 | $0.582 |
| Sol Medium | 100 | $1.365 |

원문 표는 참고값입니다. 직접 실행한 값은 모델 인증 방식, 세션 길이, 재시도 여부, 캐시 사용량에 따라 달라질 수 있습니다.
